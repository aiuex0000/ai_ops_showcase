# Ask AI

このファイルは、`docs/ai_ops_showcase/` フォルダや Git URL を AI に渡して説明してもらう時の prompt 集です。

どの prompt も、ローカルのフォルダパスでも Git URL でも使えるように書いています。

## 3分で要約して

```text
このフォルダ（または Git URL）を読んで、
何を見せるための資料なのかを3分で把握できるように要約してください。
特に、何が source-of-truth で、何が runtime surface なのかも整理してください。
```

## Web 制作が分かる人向けに説明して

```text
このフォルダ（または Git URL）を読んで、
Web 制作は分かるけれど AI 運用の裏側には詳しくない人向けに説明してください。
skills / workflows / state stack / archive / guardrails が、
作業フローの中でどう役立つかを中心にまとめてください。
```

## 自分の環境へ移植するならどうするか

```text
このフォルダ（または Git URL）を読んで、
この構成を自分の作業環境へ移植するなら、どこから真似するとよいか教えてください。
最初に導入しやすい部分と、後から足したほうがよい部分を分けて説明してください。
```

## 専門用語を抜いて初心者向けに説明して

```text
このフォルダ（または Git URL）を読んで、
専門用語をできるだけ減らしながら初心者向けに説明してください。
ただし、source-of-truth と runtime surface の違いは省かず、
日報・引き継ぎ・ログ整理の話につながるようにまとめてください。
```

## 使い方のコツ

- 最初に `README.md` を読ませると説明が安定しやすいです
- 用語で詰まったら `glossary.md` も一緒に読ませると噛み砕きやすくなります
- sample を詳しく見せたい時は、`samples/` ごと読ませるより、対象 sample を 1 本ずつ指定したほうが説明がぶれにくいです
